# Деплой проекта High-med-risk на Railway

**Важно:** модель хранения изменилась. Раньше ответы хранились только в браузере. **Теперь все ответы
сохраняются на сервере в базе данных PostgreSQL**, и их видно в админке. Это нужно для сбора заявок от всех
клиентов в одном месте и выгрузки в PDF.

## 2. Деплой на Railway

1. Зайди на [railway.com](https://railway.com) и войди в аккаунт.
2. **New Project** → **Deploy from GitHub repo** → выбери репозиторий `high-med-risk-survey`.
   Railway сам определит Node.js по `package.json`, выполнит `npm install` и `npm start`.

### 2.1. Добавить базу данных PostgreSQL

3. В проекте нажми **New** → **Database** → **Add PostgreSQL**.
4. Открой сервис приложения → вкладку **Variables**. Добавь переменную `DATABASE_URL` со ссылкой на базу:
   нажми **New Variable** → **Add Reference** → выбери Postgres → `DATABASE_URL`.
   (Так приложение получит строку подключения к базе.)

### 2.2. Задать переменные окружения

В том же разделе **Variables** добавь:

- `ADMIN_PASSWORD` — **обязательно**. Пароль для входа в админку (придумай надёжный).
- `SESSION_SECRET` — опционально. Случайная строка для подписи cookie сессии (если не задать, используется
  `ADMIN_PASSWORD`).

> SSL к базе включается автоматически для удалённого хоста. Если потребуется отключить — задай `DATABASE_SSL=false`.

### 2.3. Домен

5. **Settings** → **Networking** → **Generate Domain**. Railway выдаст ссылку вида `https://xxx.up.railway.app`.

При первом запуске приложение само создаст таблицы в базе (`submissions`, `files`).

---

## 3. Ссылки

- **Medium Risk** (отправлять клиентам): `https://<домен>/medium.html`
- **High Risk** (отправлять клиентам): `https://<домен>/high.html`
- **Админка** (только для тебя): `https://<домен>/admin.html` — вход по `ADMIN_PASSWORD`.

Это две отдельные ссылки: можно отправлять Medium одним клиентам, High — другим. Все ответы с обеих ссылок
стекаются в одну админку (с фильтром по типу).

---

## 4. Как это работает

- **Опросник** на испанском по умолчанию, с переключателем ES/EN в шапке.
- Первые два вопроса — ФИО законного представителя и Email.
- После отправки клиент видит **только экран «успешно»** — свои ответы он не видит.
- **Сервер** сохраняет ответы в PostgreSQL; загруженные документы хранятся в таблице `files`.
- **Админка** показывает все заявки, фильтр Medium/High, детальный просмотр «вопрос → ответ», скачивание
  файлов и кнопку **Download PDF** (PDF со всеми вопросами и ответами + подпись).

## 6. Безопасность (важно)

Теперь на сервере хранятся чувствительные данные KYC (имена, документы, декларации). Базовая защита:
HTTPS (Railway даёт автоматически), вход в админку по паролю, cookie `HttpOnly`/`Secure`. Если по комплаенсу
нужны шифрование «на покое» и журнал доступа — это отдельная задача поверх текущей версии.
